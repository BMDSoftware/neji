Help What's becas?

becas is a web application, API and widget for biomedical concept identification. It finds and annotates over 1,200,000 biomedical concepts in your biomedical text and publications.

Each identified concept is enriched with hyperlinks to relevant knowledge resources, where you can learn more about each concept.

Annotated concepts can be exported in various popular formats, facilitating usage in third-party applications.

You can read more about becas in the about page.

How to use

becas exposes its functionality through three interfaces.

  • This web application, where you can directly input text or PubMed articles' PMIDs for annotation;
  • An HTTP REST API that you can integrate in your own custom text processing pipeline;
  • A Javascript widget that you can embed in your pages to enhance them with text annotation capabilities.

The following sections teach you how to use this web application. For help on using the API or the widget, please refer to the dedicated documentation on their pages.

Application Tour

If you never used becas, the application tour is a good place to start. It will guide you through the main application workflow and teach you the basics, while introducing the various controls available for you to use.

Take the tour now and when you're finished return to this page and continue reading.

The Input Interface

The becas homepage, depicted bellow, is made up of two major areas:

  • The semantic group controls, on the left;
  • The text input controls, on the right.

Using the semantic group controls you can select the types of concepts you want becas to recognize and annotate. You should select at least one semantic group for annotation.

Then you can use the text input controls to give becas the text you want to annotate. The Try Sample button allows for quick testing and experimentation of the system without having to input any text. Click it multiple times to cycle through various text samples you can try.

Plain text annotation

To annotate text you can:

  • Directly type or paste text in the text-area;
  • Click the Upload file button and select a file to annotate;
  • Drag a text file or a snippet of text from your computer to the text-area.

After typing or pasting text in the text-area, click the Annotate text button to trigger the annotation process. When you upload or drag a file, annotation starts immediatelly without further input.

Notes If the Upload file button does not appear in the text input controls, your browser does not support the FileReader API.
If you can't drop files in the text-area, your browser does not support the Drag and Drop API.
You should upgrade your browser to take advantage of all available features.

PubMed abstract annotation

becas can automatically fetch PubMed abstracts from NCBI servers and annotate them for you.

Click the Annotate PubMed article button on the text input controls and a popup dialog will ask you to enter the PMID of the publication you want to annotate.

Hint You can bookmark and share the URLs of annotated articles.

The Concepts Interface

After submitting text, a file or a PMID you are presented the concepts interface, illustrated here, which is composed by four primary elements:

  • The semantic group controls, on the left;
  • The annotated text widget, on the top right;
  • The concept tree, on the bottom right;
  • The export button, bellow the annotated text, on the right.

Plain text content or PubMed articles' title and abstract are displayed in the annotated text widget, where all recognized concepts are highlighted in the colour corresponding to the concept type, or semantic group.

Terms tagged with more than one concept from different types are deemed ambiguous and highlighted with a dashed border.

You can use the semantic group controls to highlight or mute specific types of concepts, allowing you to focus only on certain entity types.

Concept Popovers

You can hover your mouse over any highlighted term to view details about identified concepts. If you're using a touch input device that does not allow hovering, just click annotated entities to open the concept popover.

For each concept, you can access external references on multiple knowledge resources. Simply click a reference link to open the corresponding resource page.

Concept Tree

All annotated terms and correspoding concepts are displayed in an interactive concept tree.

The tree structure, pictured bellow, is a simple labeled hierarchy of:

  • Semantic group (number of annotated terms)
    • Annotated entity (number of associated concepts)
      • Concept (number of external references)
        • External reference

Hint Expand and collapse whole semantic groups at a time by clicking the and icons, on the right side of the tree.

To easily locate annotated terms in the text, you can focus all occurrences of a given annotated entity in the annotated text widget by clicking the icon next to each annotated entity node on the concept tree. All entities will temporarily be muted while occurences of the focused entity are highlighted.

Just like in concept popovers, you can click any external reference to follow the link.

Exporting Results

becas exports annotated text with recognized concepts in multiple popular formats. Simply click the Export button and select your preferred format.

Each format supports different features, so you can choose the format you want depending on your needs.

JSON

becas exports annotated text in JSON format. Results exported in JSON feature sentence splitting and concept identification data.

JSON exports include concepts nested to any depth level.

XML

becas exports results in IeXML. Results exported in XML include the same information as JSON exports (sentence splitting and concept identification), but nested annotations are limited to a depth of one, with deeper annotations being resolved to the largest span.

Note PubMed articles are fetched from NCBI servers in MEDLINE XML format (<PubmedArticleSet>) and made available for export only in the same format, with the <ArticleTitle> and <Abstract> fields enriched with IeXML annotations.

A1

A1 is a standoff format used in many text-mining research tasks and in popular applications like brat.

becas exports annotations in A1 format, including concept identification data. A1 exports include concepts nested to any depth level.

CoNLL

Of all exported formats, CoNLL is the most verbose and comprehensive.

Results exported by becas in CoNLL format include, in addition to concept identification, natural language processing information, like sentence splitting, tokenization, lemmatization, part-of-speech (POS) tagging and chunking.

CoNLL exports include concepts nested to any depth level.

Detailed information about each exported format, as well as a comparison between them is available in the API documentation.

Hint If you find yourself using the export functionality a lot, maybe you should consider using the HTTP API to automate your workflow.
